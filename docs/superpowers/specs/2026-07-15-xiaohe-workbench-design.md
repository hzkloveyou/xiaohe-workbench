# 小贺的工作台：产品与技术设计

## 1. 产品定位

“小贺的工作台”是面向所有访客的中文个人启动台。访客无需注册即可使用搜索、书签和今日专注功能；注册后可在不同设备之间同步数据。网站部署在 GitHub Pages，Cloudflare 提供 API、D1 数据库、DNS 和自定义域名服务。

主域名为 `https://080492.xyz`，API 使用 `https://api.080492.xyz`。`www.080492.xyz` 重定向至主域名。

## 2. 成功标准

- 首次打开即可使用，不用先理解配置流程。
- 三个核心模块覆盖高频操作：搜索、导航、今日专注。
- 访客可添加书签、拖动卡片、调整布局并切换主题。
- 游客数据保存在本机；登录后数据可跨设备同步。
- 桌面、笔记本、平板和手机均有专门布局，不出现横向滚动。
- 动效清新、克制，不妨碍阅读，并完整支持减少动态效果模式。
- 通过类型检查、Lint、单元测试、构建和浏览器端验收。

## 3. 第一版功能范围

### 3.1 智能搜索与指令栏

- 首页中心搜索框，支持键盘快捷键 `Ctrl/Cmd + K` 聚焦。
- 自动区分搜索词和完整网址。
- 可切换百度、必应、Google 等搜索引擎，并记住个人选择。
- 支持快捷前缀，例如 `gh react`、`mdn grid`。
- 最近搜索只保存在用户自己的数据中，可随时清除。

### 3.2 快捷导航与书签

- 默认提供精简的中文高频分类：AI、工作、开发、学习、影音。
- 支持添加、编辑、删除、分组、搜索和拖动排序。
- 不依赖不稳定的远程图片；网站图标优先使用本地 SVG、文字首字和统一图标集。
- 空状态提供明确的添加入口。
- 可导出和导入完整配置备份。

### 3.3 今日专注

- 今日三件事：最多突出三项，其余任务折叠显示。
- 快速便笺：自动保存。
- 25 分钟专注计时器：开始、暂停、重置和完成反馈。
- 计时器状态在刷新后恢复；浏览器允许时显示完成通知。

### 3.4 个性化

- 书签和首页卡片支持鼠标、触摸和键盘排序。
- 主题包含晨雾、暮光、夜雾和跟随系统；晨雾为默认主题。
- 卡片可显示、隐藏和调整顺序。
- 导入、导出与恢复默认配置。

### 3.5 账户与跨设备同步

- 游客模式始终可用，注册不是使用前提。
- 账户使用用户名和密码；密码不以明文保存。
- 注册时生成一次性展示的恢复码，服务器只保存恢复码哈希。
- 登录成功后使用安全、HttpOnly、Secure、SameSite Cookie 会话。
- 同步以本地优先方式工作：离线时继续操作，恢复网络后自动重试。
- 每个实体记录更新时间和删除标记；发生冲突时按实体合并，避免整份配置互相覆盖。
- 服务端进行来源校验、输入校验、登录限流和统一错误处理。

## 4. 视觉与交互设计

采用已确认的 A 方案“晨雾玻璃”：天空蓝、薄荷绿和奶油白为主色，半透明卡片、低对比度边框、柔和阴影和统一圆角构成层级。

- 背景使用 2–3 个缓慢移动的模糊渐变光斑，不使用密集粒子。
- 首次进入时标题、说明、搜索框和卡片分层淡入，总时长控制在约 600 毫秒内。
- 卡片悬停最多上移 6 像素，桌面端可有不超过 3 度的轻微倾斜；触屏端关闭倾斜。
- 按钮覆盖默认、悬停、按下、聚焦、加载和禁用状态。
- 所有持续动画仅使用 `transform` 和 `opacity`，并在减少动态效果模式下关闭。
- 界面是工作台而非产品官网，因此不加入营销 Hero、案例数据或虚构用户评价。

## 5. 信息架构与响应式布局

桌面端采用顶部品牌区、中心搜索区和可拖动卡片网格。导航书签占主要宽度，今日专注与账户/设置为辅助列。

- 大屏：12 列网格，主内容最大宽度 1440px。
- 普通笔记本：8 列网格，保持双栏结构。
- 平板：4 列网格，搜索和导航优先，辅助卡片下移。
- 手机：单列信息流，底部或抽屉式设置入口，触控目标不小于 44px。

## 6. 技术架构

### 6.1 前端

- React、TypeScript、Vite。
- CSS 变量与模块化样式构建设计令牌；避免大量内联样式。
- Motion 提供克制的进入和布局动画。
- dnd-kit 提供鼠标、触摸和键盘可访问的拖拽排序。
- IndexedDB 保存本地工作台数据和待同步变更。
- PWA 清单与 Service Worker 提供安装和离线壳。

前端按领域拆分为搜索、书签、今日专注、个性化、认证和同步模块。通用按钮、卡片、弹窗、表单、空状态、错误状态和动效容器独立封装。

### 6.2 后端

- Cloudflare Worker 提供 REST API。
- Cloudflare D1 保存用户、会话、恢复码、同步实体和登录限流数据。
- Web Crypto 负责密码派生、令牌哈希和安全随机数。
- API 仅允许主域名和本地开发源，所有写操作校验 `Origin`。

### 6.3 数据流

1. 用户操作首先写入本地 IndexedDB，并立即更新界面。
2. 已登录时，同步队列把变更批量提交给 API。
3. 服务端按用户和实体版本应用变更，返回新的服务端版本。
4. 登录或重新联网时拉取远端增量，按实体更新时间与删除标记合并。
5. 同步失败不回滚本地操作；界面显示非阻塞状态并指数退避重试。

## 7. 数据与安全

- 用户名规范化后唯一，密码最少 12 个字符。
- 密码使用带独立随机盐的 PBKDF2-SHA-256 派生，参数集中管理并可升级。
- 会话令牌和恢复码只以 SHA-256 哈希形式存储。
- 会话默认 30 天过期；退出登录立即撤销当前会话。
- D1 查询全部参数化；请求体设置大小上限。
- 不记录密码、恢复码、Cookie 或工作台正文到日志。
- 不使用第三方广告或行为追踪；访问统计若后续启用，只采用匿名聚合数据。

## 8. 错误和边界状态

- 网络离线：保留全部本地能力，明确显示“离线，等待同步”。
- 认证失效：停止上传并提示重新登录，不删除本地数据。
- 同步冲突：自动按实体合并；无法自动合并的便笺保留本地与远端两个版本供恢复。
- 数据损坏：拒绝导入并指出具体问题，当前数据不被覆盖。
- API 错误：显示可重试提示，不暴露服务端堆栈。
- 空数据、加载、禁用和提交中状态均有独立视觉反馈。

## 9. 测试与验收

- 单元测试：URL/搜索解析、数据合并、主题选择、计时器恢复、密码与令牌工具。
- 组件测试：书签编辑、任务操作、登录表单、错误与空状态。
- API 测试：注册、登录、退出、恢复、授权、限流和同步冲突。
- 端到端测试：首页首次使用、游客配置、账户同步、拖拽、移动端布局和离线恢复。
- 静态检查：TypeScript 严格模式、ESLint、生产构建。
- 浏览器检查：桌面、平板、手机、键盘操作、控制台错误、横向滚动和减少动态效果。

## 10. 部署

- GitHub Actions 构建前端并发布到 GitHub Pages。
- GitHub Actions 或 Wrangler 发布 Worker 和 D1 迁移。
- Cloudflare DNS 将根域名与 `www` 指向 GitHub Pages，将 `api` 子域绑定到 Worker。
- 发布后从公网验证 HTTPS、PWA、注册登录、游客模式、跨设备同步和退出登录。

## 11. 后续扩展边界

第一版不加入天气、日历、邮箱、AI 助手、多人共享或第三方 OAuth。模块注册表、数据版本和同步协议保留扩展能力，后续可以在不破坏现有用户配置的前提下新增卡片。
